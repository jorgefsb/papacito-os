## 📊 PRINCIPIO #2: Framework Leverage/Speed/Energy

Evaluar TODA idea nueva contra las 3 métricas de Jorge:

### Scoring:

| Métrica | Pregunta | Score Alto (prioriza) | Score Bajo (skip) |
|---------|----------|----------------------|-------------------|
| **Leverage** | ¿Esto me da leverage? ¿Multiplica esfuerzos? | 8-10/10 | 1-4/10 |
| **Speed** | ¿Esto me da speed? ¿Acelera resultados? | 8-10/10 | 1-4/10 |
| **Energy** | ¿Esto me da energía? ¿Es sostenible? | 7-10/10 | 1-4/10 |

### Decisión Matriz:

**Score Total = Leverage + Speed + Energy (max 30)**

- **24-30:** DO IT NOW (las 3 altas)
- **18-23:** PROBABLY DO IT (2/3 altas)
- **12-17:** CONSIDERAR (solo si Core 4 oriented)
- **0-11:** SKIP (low ROI)

### Ejemplos:

**GOOD Decision (Score 25):**
- Obtener métricas AGA para caso de éxito Brandcade
- Leverage: 10/10 (desbloquea pipeline completo)
- Speed: 10/10 (blocker crítico)
- Energy: 5/10 (requiere follow-up)
- **Total: 25 → DO IT NOW**

**BAD Decision (Score 7):**
- Implementar 34 agentes técnicos completos
- Leverage: 3/10 (mayoría no útiles)
- Speed: 1/10 (6-7 hrs implementación)
- Energy: 3/10 (mantenimiento alto)
- **Total: 7 → SKIP**

## 🚫 PRINCIPIO #3: Anti-Overkill Check

**ANTES de agregar CUALQUIER cosa nueva, pregunta:**

### 6 Preguntas Anti-Overkill:

1. ❓ **¿Esto ayuda a alguna de las 4 Responsabilidades de CEO/GM?**
   - Traer Dinero (Core 4)
   - Atraer/Retener Talento
   - Estrategia para Competir
   - Escribir la Historia
   - Si NO → CUESTIONAR seriamente

2. ❓ **¿Esto ayuda DIRECTAMENTE a vender más? (Core 4)**
   - Si NO → CUESTIONAR (probablemente overkill)

3. ❓ **¿Ya tengo algo que hace esto (overlap)?**
   - Si SÍ → SKIP (redundante)

4. ❓ **¿El ROI justifica el tiempo de implementación?**
   - Si NO → CUESTIONAR (low ROI)

5. ❓ **¿Es estratégico (mi rol) o táctico (del equipo)?**
   - Si táctico → Team lo hace, no tú

6. ❓ **¿Es Core 4 oriented?**
   - Core 4: Warm Outreach, Cold Outreach, Post Content, Paid Ads
   - Si NO → Lower priority (pero puede servir a otras 3 responsabilidades)

### Señales de Overkill (RED FLAGS):

🚩 "Sería cool tener..."
🚩 "Por si acaso en el futuro..."
🚩 "Todos los demás tienen..."
🚩 "Solo faltan otros 20 agentes..."
🚩 "Necesito este feature para completar el sistema..."

**Cuando veas esto → STOP. Vuelve a Core 4.**

## 📈 PRINCIPIO #4: Priorización por Impacto en Ventas

### Tier 1 - CRÍTICO (hacer AHORA):
- Genera leads DIRECTAMENTE
- Cierra deals DIRECTAMENTE
- Desbloquea ventas (e.g., caso de éxito)

**Ejemplos:**
- Obtener métricas AGA (desbloquea ventas Brandcade)
- BD Outreach Flow → Cerrar 3 marcas Q1
- LinkedIn automation → Leads directos

### Tier 2 - IMPORTANTE (hacer PRONTO):
- Mejora eficiencia de Core 4
- Genera visibility → leads indirectos
- Optimiza procesos de venta

**Ejemplos:**
- Usar PapacitoOS con requests reales (validación)
- Sales Machine automation para Sparkplug
- Compartir template open source (thought leadership)

### Tier 3 - NICE TO HAVE (hacer DESPUÉS):
- Mejora calidad pero no impacta ventas directamente
- Optimizaciones internas
- "Sería útil pero no urgente"

**Ejemplos:**
- Phase 2 agentes selectos (después de validación)
- Poblar GitHub repos (long-term asset)
- Documentación avanzada

### Tier 4 - SKIP (overkill / no prioritario):
- No relacionado con ventas
- Overkill / over-engineering
- Rol táctico (no estratégico)

**Ejemplos:**
- Implementar 34 agentes completos
- Engineering agents (frontend, backend, etc.)
- Testing agents (api-tester, etc.)
- Design agents hands-on

## 🎯 PRINCIPIO #5: Strategic vs Tactical

### Pregunta clave:
**¿Esto es STRATEGIC (mi rol GM/CEO) o TACTICAL (mi equipo)?**

| Strategic (SÍ hacer) | Tactical (NO hacer, equipo lo hace) |
|---------------------|--------------------------------------|
| Decisiones de negocio | Coding hands-on |
| Strategy de BD/ventas | Design hands-on |
| Product vision | Testing/QA |
| Content strategy | Implementación técnica |
| Hiring decisions | Execution táctica |
| Prioritization | Tasks operativas |

**Tu rol:** GM Amber + CEO Sparkplug = Strategic level

**NO es tu rol:** Coder, designer, QA tester = Tactical level

**Decisión:**
- Si es Strategic → Puede ser tu trabajo
- Si es Tactical → Delega al equipo

## ⚖️ PRINCIPIO #6: Opportunity Cost

**Pregunta:** Si hago esto, ¿qué NO estoy haciendo?

### Framework:

**Opción A:** Implementar 34 agentes (6-7 hrs)
**Opción B:** Usar ese tiempo en BD Outreach Flow → Cerrar 1 deal ($15-35K)

**Opportunity cost de A:** Perder $15-35K en revenue
**Decisión:** B tiene mejor ROI

### Ejemplo Real:

**Opción A:** Crear sistema custom de analytics (20 hrs)
**Opción B:** Usar Google Analytics existente (1 hr setup) + 19 hrs en Core 4 outreach

**Opportunity cost de A:** 19 hrs que podrían generar ventas
**Decisión:** B tiene mejor ROI

```
   ↓ Mejor ROI

   ↓

```
